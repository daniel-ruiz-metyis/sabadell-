# Banco Sabadell · Cuadro de mando de renovación (MVP)

A five-screen operations dashboard replicating the card-renewal mockups, built as
a React front end with a deliberately simple Python (FastAPI) back end. Every
figure is driven by editable CSV files, so you can download the data, change it in
Excel, and upload it back to reshape the dashboard.

## Screens

1. **Proceso de renovación** — funnel KPIs, waterfall by tarjeta type, breakdown table.
2. **Estampación** — per-estampador KPIs and the lot-level stamping table.
3. **Tarjetas activadas** — activation curve and spend-vs-prior-month histograms.
4. **Tarjetas bloqueadas** — block reasons/origin, client view, segment pie, territory.
5. **Comunicaciones operativas** — channel KPIs, milestone table, interaction funnel.

## Filters (fully wired)

- **Año / Mes caducidad** — real filters. Selecting a year/month re-queries the
  backend (`GET /api/data/{name}?anio=...&mes=...`) and every page re-renders with
  that period's numbers. Ten periods ship out of the box: 2024–2025 quarterly
  (ENE/ABR/JUL/OCT) plus 2026 ENE/ABR (2026 stops at ABR — later months are in the
  future relative to the dashboard's reference date and have no actuals).
- **Fecha de consulta** — derived per period, mirrors the ~56-day reporting lag
  seen in the source mockup (caducidad 30/04/2026 → consulta 25/06/2026).
- **Segmento / Tipología tarjeta** — real client-side filters on any dataset that
  carries those dimensions (today: the Renovación breakdown table and funnel).
  Toggling a chip recomputes totals, percentages, and MoM trend directly from the
  filtered rows.

## Running

Two processes. From the `Dashboard` folder:

### 1. Backend (port 8000)

```bash
cd backend
python -m venv .venv
# Windows PowerShell:  .venv\Scripts\Activate.ps1
# macOS/Linux:         source .venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

### 2. Frontend (port 5173)

```bash
cd frontend
npm install
npm run dev
```

Open http://localhost:5173. The dev server proxies `/api` to the backend.

## Editing the data

- **Download** (filter bar, top right) grabs a zip of every CSV in `backend/data`.
- Edit any CSV in Excel. Keep the header row and the file name unchanged
  (e.g. `desglose.csv`). Most datasets carry `anio`/`mes` columns — add a row per
  period you want to appear as a selectable combo.
- **Subir / Upload** a single CSV to replace that dataset. The dashboard reloads
  and re-reads the available year/month combos.

Numbers use plain integers in the CSVs (no thousands separators); the UI formats
them in Spanish locale (45.681, 74%). See `backend/data/` for the full schema of
each dataset, and `backend/generate_data.py` for how the multi-year mock data was
derived from the single real baseline (2026-ABR, taken verbatim from the source
mockups) — rerun it any time to reshuffle the other periods.

## Notes

- Territory is shown as a ranked bar of the top autonomous communities rather than
  a choropleth map, to keep the MVP dependency-free.
- `combos.csv` is the source of truth for which (año, mes) pairs are selectable;
  it's regenerated by `generate_data.py` alongside the rest of the mock data.
